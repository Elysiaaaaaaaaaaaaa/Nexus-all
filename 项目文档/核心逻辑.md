# 核心逻辑

```mermaid
flowchart TD
  A[用户输入需求] --> B[imagination: 记录idea]
  B -->|确认| C[outline: 生成大纲]
  C --> D[自动切换到screen, now_state=pending]
  D -->|确认| E[screen: 生成分镜脚本]
  E --> F{需要修改分镜?}
  F -->|需要修改| G[modify: 按modify_num修订分镜]
  G --> E
  F -->|不需要| H[animator: now_state=pending]
  H -->|确认| I[生成视频]
  I --> J{视频是否生成完?}
  J -->|未完成| H
  J -->|完成| K[end_session/chat_with_assistant=false]
```

## 状态说明（text2video）
- outline 生成后：`now_task=screen`，`now_state=pending`，等待确认生成分镜脚本。
- screen 生成后：`now_state=modify_comfirm`，仅此阶段弹出修改确认。
- animator 生成后：`now_state=pending`，点击确认继续生成下一段视频。
- 当 `video_generating >= len(screen)` 时，设置 `chat_with_assistant=false` 并结束会话。

## 图生视频流程（image2video）

```mermaid
flowchart TD
  A[用户输入需求] --> B[imagination: 记录idea]
  B -->|确认| C[outline: 生成大纲]
  C --> D[figure_design: 角色描述/默认角色]
  D --> E[story_board_prompting: 生成首帧提示词]
  E --> F{是否有参考图?}
  F -->|有| G[i2i生成首帧图片]
  F -->|无| H[t2i生成首帧图片]
  G --> I[script: 生成视频提示词]
  H --> I
  I --> J[animator: 图生视频生成]
```
